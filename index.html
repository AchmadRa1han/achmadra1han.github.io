Materi 1: Strategi Testing

Software testing adalah proses mengevaluasi perangkat lunak untuk menemukan cacat (bug) dan memastikan produk bekerja sesuai kebutuhan. Tujuannya adalah untuk menjamin kualitas, keamanan, dan meningkatkan pengalaman pengguna. Proses ini mengikuti siklus hidup (STLC) yang terstruktur, mulai dari Test Planning (perencanaan), Test Design (perancangan kasus uji), Test Execution (pelaksanaan), hingga Pelaporan. Testing dapat diklasifikasikan berdasarkan struktur (Black-box, White-box) atau berdasarkan level (Unit, Integration, System, Acceptance).

Materi 2: UI/UX Testing

Perbedaan mendasar antara UI dan UX testing adalah fokusnya. UI (User Interface) Testing berfokus pada tampilan visual aplikasi, seperti warna, ikon, layout, dan konsistensi. Tujuannya adalah memastikan tampilan benar dan responsif di berbagai perangkat. UX (User Experience) Testing berfokus pada pengalaman pengguna secara keseluruhan, menguji apakah alur kerja aplikasi mudah digunakan, logis, dan efisien. Fokus utama UX testing meliputi Usability (kegunaan), Aksesibilitas (untuk penyandang disabilitas), dan Alur Kerja (workflow) pengguna.

Materi 3: Testing Plan

Testing Plan adalah dokumen formal yang menjadi panduan dan menjelaskan bagaimana proses pengujian akan dilakukan. Dokumen ini mencakup ruang lingkup pengujian, strategi yang dipakai, sumber daya (tim, alat), dan jadwal. Komponen penting dalam Test Plan meliputi: Test Items (apa yang akan diuji), Features to be Tested (fitur yang diuji), Features not to be Tested (fitur yang tidak diuji), Approach (pendekatan, misal manual atau otomatis), Item Pass/Fail Criteria (kriteria lulus/gagal), dan Approvals (persetujuan).

Sumber: Kelompok 3

Materi 4: Test Scenario, Test Case, & Bug Report

Ini adalah tiga elemen penting dalam dokumentasi pengujian. Test Scenario adalah gambaran umum tentang *apa* yang harus diuji (misal: "Periksa fungsi login"). Test Case adalah langkah-langkah detail tentang *bagaimana* melakukan pengujian, yang mencakup ID, precondition, test steps, test data, dan expected result. Jika pengujian gagal (Actual Result tidak sama dengan Expected Result), maka dibuatlah Bug Report. Ini adalah laporan formal kesalahan yang berisi detail masalah, langkah-langkah untuk mereproduksi bug (steps to reproduce), serta tingkat Severity (dampak bug) dan Priority (urgensi perbaikan).

Materi 5: Unit Testing

Unit Testing adalah pengujian yang berfokus pada unit-unit terkecil dan terisolasi dalam sebuah kode, seperti function atau method. Ini adalah level pengujian paling awal dan biasanya dilakukan oleh developer. Tujuannya adalah untuk mendeteksi bug sedini mungkin, yang pada akhirnya menghemat waktu dan biaya. Pola dasar yang populer digunakan adalah AAA (Arrange, Act, Assert): Arrange (menyiapkan kondisi awal), Act (menjalankan fungsi yang diuji), dan Assert (memverifikasi hasilnya). Framework yang umum digunakan adalah JUnit (Java), Pytest (Python), dan Jest (JavaScript).

Materi 6: API Testing

API (Application Programming Interface) Testing adalah proses pengujian yang dilakukan langsung pada lapisan API untuk memastikan fungsionalitas, keandalan, keamanan, dan performa. Pengujian ini penting untuk memvalidasi logika bisnis dan memastikan komunikasi antar sistem berjalan lancar. Proses dasarnya adalah mengirimkan Request (permintaan) ke sebuah URL endpoint menggunakan method (seperti GET, POST, PUT), kemudian memvalidasi Response (balasan) dari server, terutama Status Code (misal 200 OK atau 404 Not Found) dan data yang dikembalikan. Tools yang sangat populer untuk API testing adalah Postman dan SoapUI.

Materi 7: Pengantar Selenium

Selenium adalah sebuah framework *open-source* yang digunakan untuk otomatisasi browser. Selenium paling sering digunakan untuk menguji aplikasi web secara otomatis. Komponen intinya adalah Selenium WebDriver, yang berfungsi sebagai penghubung (jembatan) antara kode skrip pengujian (yang bisa ditulis dalam bahasa Java, Python, C#, dll.) dengan browser (seperti Chrome, Firefox, Edge). Skrip ini dapat memberi perintah pada browser untuk melakukan aksi pengguna, seperti membuka URL, mengklik tombol, mengisi form, dan memvalidasi teks di halaman.

Materi 8: Pengantar Cypress

Cypress adalah framework end-to-end testing modern yang khusus dirancang untuk menguji aplikasi web (terutama yang dibangun dengan React, Vue, Angular, dll.). Berbeda dari Selenium, Cypress memiliki beberapa keunggulan unik, seperti Test Runner interaktif yang memungkinkan kita melihat eksekusi tes secara visual dan real-time. Fitur unggulan lainnya adalah "Time Travel" (melihat snapshot dari setiap langkah tes untuk debugging) dan Automatic Waits (Cypress otomatis menunggu elemen muncul sebelum berinteraksi, sehingga tes lebih stabil). Perintahnya juga intuitif, seperti `cy.visit()` untuk membuka URL dan `cy.get()` untuk memilih elemen.
